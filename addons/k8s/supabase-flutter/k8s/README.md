# Kubernetes (optional add-on) — {{ project_name }}

> **Optional and not free-tier.** A cluster + managed Postgres are not free.
> This scaffold is here when you outgrow the one-button Cloudflare deploy
> (`make deploy`). It is **not** generated by default — re-run firestarter with
> `--set include_k8s=yes` (or answer `yes` at the prompt) to include it.

Kustomize manifests for the API layer (PostgREST + GoTrue).

```
k8s/
├── base/                 # PostgREST + GoTrue Deployments/Services + namespace
└── overlays/
    ├── staging/
    └── production/       # +ingress
```

## Apply

```bash
kubectl apply -k k8s/overlays/staging      # or .../production
```

## What's in / out of cluster
- **In cluster:** PostgREST, GoTrue (stateless API).
- **Out of cluster (bring your own):** **Postgres/PostGIS** — use a managed
  instance; provide its URL via the Secret (`base/secret.example.yaml`). Don't
  run the stateful DB from this scaffold.
- **Splash page:** it's a static Vite build — host it on object storage / a CDN
  (or behind the same ingress as a static bucket), not as a k8s Deployment.
- **Flutter app:** ships to app stores / served as a web build from a CDN.

**Never commit real secret values** — use sealed-secrets / external-secrets.
